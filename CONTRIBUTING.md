# Contributing

Contributions to ExamFlowOS are welcome.

## How to contribute
- improve documentation
- suggest product ideas
- refine architecture and design notes
- help structure the study experience more clearly

## Guidelines
- be respectful and constructive
- keep changes focused and clear
- document significant decisions
- prefer thoughtful iteration over unnecessary complexity

## Pull request expectations
- explain the change clearly
- link related context when helpful
- keep the repository consistent with its public product vision
